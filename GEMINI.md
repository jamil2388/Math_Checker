# Project: Resume Builder

## General Instructions

- During the beginning of the conversation, always check the MILESTONES.md, README.md files to stay up to date 
with the progress of the project
- Before executing your ideas, always articulate the ideas with me. If I give you the signal,
then go ahead for execution

## Coding Style

- Always come up with a generalized model for the code. For example - If you are suggesting a handful of function calls and couple of functionalities,
make sure to design them in a modular way such that, they can be easily expandable to future cases, and also they follow OOP
- Always create an elaborate comment for the start of a function call. That comment will document all the arguments, types, the function it does,
and the output types it returns
- Always do Test Driven Development. Make the tests, code your solution to pass the tests. Share your ideas with me before coding the test solutions
- Also create comments whenever the line of code might get confusing for the user. But don't overdo the commenting
- Start the comment with a capital letter, and write the comment as if I would have writen that (From my POV)
- Always try to make the functions modular, such that each function is designed to focus on one job, not multiple
- Whenever there are multiple spots where a piece of code repeats itself anywhere in the codebase, you can 
write a housekeeping or util function for that, and then just call that to reduce redundancy. Please notify me whenever you do that
- We are following OOP, so whenever there is a scope of generalizing a task and forming a class and its objects, do so
